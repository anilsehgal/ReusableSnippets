Duplicate Primary Key Action

Duplicate Primary Key Action setting contains the possible options to be picked up if while inserting, a duplicate primary key is encountered. Following options are available to user:

  • Reject Row: This rejects the duplicate row and continues with the import process
  • Stop Import: (Not Recommended) This immidiately terminates the import process after the duplicate key is encountered, this can lead to insertion of incomplete rows
  • Fire Update: This fires an update if a duplicate primary key is encountered
